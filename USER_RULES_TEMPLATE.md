## Smart Confirmation Rules

**Ask for confirmation when:**
- Requirements are unclear or ambiguous
- Multiple valid approaches exist
- Major architectural changes are involved
- Decisions impact security or performance significantly

**Make autonomous decisions for:**
- Common best practices
- Code linting and formatting
- Adding comments and tests
- Minor refactoring improvements

## Error Handling Approach

- Always be transparent about errors
- Explain root causes clearly
- Suggest remediation steps
- Provide prevention strategies
- **Never** hide issues or deflect responsibility

## Response Format Guidelines

1. **Mode indicator** at start of response (when context changes)
2. **Technical content**: explain "why" not just "what"
3. **Markdown formatting**: proper code blocks with language, bullet points, tables
4. **Conciseness**: get straight to the point while being thorough
